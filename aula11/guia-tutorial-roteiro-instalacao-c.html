Instalação do Ambiente GCC/MinGW no Windows + VS Code

Tutorial limpo e direto para configurar C no Windows usando GCC, GDB e Visual Studio Code.

Visão Geral

Você irá configurar:

  • Compilador gcc (linguagem C)
  • Depurador gdb
  • Variável de ambiente PATH
  • Extensões do VS Code
  • Arquivos .vscode/tasks.json e .vscode/launch.json

1. Download do GCC / MinGW (WinLibs)

1
Acesse o link de download informado (download inicia automaticamente):
WinLibs MinGW - Download
2
Baixe o arquivo compactado (.zip ou .7z).
3
Descompacte o conteúdo em C:\mingw32 (ou C:\mingw).
4
Confirme se existe a pasta:
C:\mingw32\bin
Dentro de bin devem existir arquivos como gcc.exe, g++.exe, gdb.exe.

2. Configurar Variável de Ambiente PATH

  1. Pressione Win + R
  2. Digite sysdm.cpl
  3. Aba Avançado → Variáveis de Ambiente
  4. Em Path, clique em Editar
  5. Adicionar:
C:\mingw32\bin

Feche e abra o terminal novamente.

Teste no Prompt / PowerShell

gcc --version
gdb --version

3. Instalar Visual Studio Code

Baixe e instale o VS Code.

Depois abra o VS Code e instale as extensões:

  • C/C++ Extension Pack (Microsoft)
  • C/C++ DevTools (Microsoft)

Link oficial da extensão:

https://github.com/microsoft/vscode-cpptools

4. Criar Projeto

1
No VS Code clique em File → Open Folder
2
Crie uma pasta, exemplo:
D:\Projetos\C
3
Dentro dela crie a pasta .vscode
4
Dentro de .vscode crie os arquivos:
tasks.json e launch.json

5. tasks.json

{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "build C",
      "type": "shell",
      "command": "gcc",
      "args": [
        "-g",
        "-Wall",
        "${file}",
        "-o",
        "${fileDirname}/${fileBasenameNoExtension}.exe"
      ],
      "group": {
        "kind": "build",
        "isDefault": true
      },
      "problemMatcher": ["$gcc"],
      "presentation": {
        "echo": false,
        "reveal": "never",
        "focus": false,
        "panel": "shared",
        "showReuseMessage": false,
        "clear": false,
        "close": true
      }
    }
  ]
}

6. launch.json

{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug C (MinGW)",
      "type": "cppdbg",
      "request": "launch",
      "program": "${fileDirname}/${fileBasenameNoExtension}.exe",
      "args": [],
      "stopAtEntry": false,
      "cwd": "${fileDirname}",
      "environment": [],
      "externalConsole": false,
      "MIMode": "gdb",
      "miDebuggerPath": "C:/mingw32/bin/gdb.exe",
      "preLaunchTask": "build C"
    }
  ]
}

7. Primeiro Programa

Crie main.c:

#include <stdio.h>
#include <stdlib.h>

int main(){
    printf("Ola, mundo!\n");
    return EXIT_SUCCESS;
}

F5 para compilar e depurar.

Ctrl + F5 para executar sem depuração.

8. Alternativa Simples

Instalar Code::Blocks, que já acompanha um subconjunto do GCC para Windows.

Depois também é possível apontar o VS Code para essa instalação.

9. Problemas Comuns

  • gcc não reconhecido: PATH não configurado.
  • gdb não conecta: caminho errado em miDebuggerPath.
  • Antivírus bloqueando: adicionar exceção para pasta do compilador.
  • F5 só compila: revisar launch.json.
